# Инструкция для агентов-переводчиков

> Этот файл — копия из шаблона `core/templates/ru/`, помещённая в `result/<имя>/translation/` при инициализации книги. Изменения, специфичные для конкретной книги, дописывай в `DOMAIN.md` рядом — не модифицируй общий гайд.

Этот файл — **обязательное чтение** перед началом любой задачи по переводу. Если тебе поручили перевести раздел, термин, упражнение, подпись к рисунку — выполняй эти правила без отклонений.

---

## 1. Что тебе дают на входе

Главный агент (или человек) выдаёт тебе задание вида:

> «Переведи раздел **N.M Section Title**. Диапазон в `extracted/text.txt`: строки 1800–2100. Диапазон страниц: `extracted/pages/page-0028.png` – `extracted/pages/page-0035.png`. Запиши результат в `docs/chapters/<NN-slug>/section-N-M.md`.»

Все пути — относительно корня папки книги (`result/<имя>/`). Если каких-то данных нет — **остановись и попроси их**, не угадывай.

Дополнительно у тебя всегда есть:

- `translation/GLOSSARY.md` — единый словарь терминов. **Использовать только эти варианты перевода.** Если термина нет — добавь его в конец файла (раздел «Pending review»), не меняя существующих.
- `translation/chapter_map.json` — карта глав/разделов (диапазоны строк и страниц).
- `translation/DOMAIN.md` — специфичные для этой книги правила (формулы, единицы, имена). Читай его перед `GLOSSARY.md`.
- `extracted/pages/page-NNNN.png` — сканы для сверки формул, рисунков, таблиц.
- `extracted/images/img-NNN.{png,jpg}` — извлечённые иллюстрации.
- `book.yml` — метаданные книги (язык, лицензия, parallelism).

## 2. Алгоритм работы

1. **Прочитай задание** и убедись, что понял границы раздела.
2. **Прочитай `DOMAIN.md`** — если в книге свои правила оформления (формулы, нотация), они там.
3. **Прочитай оригинал**: сначала `extracted/text.txt` в указанном диапазоне строк, затем — все соответствующие `extracted/pages/page-*.png`. **Сканы важнее текста**: `pdftotext` теряет нижние/верхние индексы, разрывает таблицы, путает порядок колонок.
4. **Сверь термины с `GLOSSARY.md`**. Выпиши незнакомые — добавишь в конце.
5. **Переведи** по правилам ниже. Старайся за один проход — потом перечитай и отредактируй на чистый русский.
6. **Сверь со сканами**: каждая формула, каждое число, каждая подпись к рисунку — должны совпадать с PNG.
7. **Запиши результат** в указанный файл. Если файла нет — создай (предварительно убедись, что родительская папка существует).
8. **Обнови `translation/PROGRESS.md`**: поставь `[~]` напротив раздела (черновик готов), укажи дату и автора (`Claude/<модель>`). Ревьюер переведёт в `[r]`, сборщик — в `[x]`.
9. **Обнови `translation/GLOSSARY.md`**, если ввёл новые термины. Каждый новый термин — отдельной строкой в формате таблицы, в разделе «Pending review».
10. **Не запускай `mkdocs build`** сам — это делает агент-сборщик. Но если правил `mkdocs.yml` — обязан проверить, что страница рендерится.

## 3. Стиль перевода

**Тон**: академический, нейтральный, без разговорных оборотов. Без англицизмов там, где есть русский термин.

**Что переводим, а что нет:**

| Категория | Действие |
|-----------|----------|
| Основной текст | Переводим. |
| Названия глав и разделов | Переводим, в скобках курсивом — оригинал при первом появлении: `## N.M Название раздела *(Section Title)*`. |
| Термины при первом введении | Русский термин **жирным**, в скобках — английский курсивом: **термин** (*term*). |
| Имена людей | Транслитерация по сложившейся традиции. При первом упоминании в скобках оригинал. |
| Названия журналов, книг, законов с именами | По устоявшейся практике (см. `DOMAIN.md`). |
| Единицы измерения | По ГОСТ 8.417 (если в `DOMAIN.md` не указано иное). |
| URL, DOI, ISBN, идентификаторы | НЕ трогаем. |
| Подписи к фигурам/таблицам/примерам | Переводим, нумерацию сохраняем. |
| Числа | Десятичная **запятая** (`3,14`), разряды через неразрывный пробел (`1 000 000` или `&nbsp;`). |
| LaTeX-числа | $1{,}5 \times 10^{23}$ (запятая через `{,}` чтобы не было лишнего пробела). |

**Не делай:**

- кальки («валидировать гипотезу», «дискачать», «паттерн») — пиши «проверить», «загрузить», «закономерность»;
- дословный перевод идиом — переписывай мысль;
- замену оригинальных обозначений переменных;
- «улучшений» текста: добавление от себя пояснений, аналогий, шуток — **запрещено**, перевод не комментирует оригинал;
- сокращений типа «т.к.», «т.е.» — пиши полностью.

## 4. Формулы, единицы, специальная нотация

Базовые правила (доменные дополнения — в `DOMAIN.md`):

- Математика — через `pymdownx.arithmatex`: `$inline$` и `$$display$$`.
- Греческие буквы и спецсимволы — через LaTeX (`$\Delta$`, `$\lambda$`), не Unicode.
- Степени и индексы в обычном тексте — только через LaTeX: «концентрация $10^{-3}$ моль/л», не «10⁻³».
- Единицы рядом с числом — неразрывный пробел: `25{,}0\ \text{°C}` в LaTeX или `25,0&nbsp;°C` в обычном тексте.

Если книга техническая (химия/физика/математика) — `DOMAIN.md` опишет дополнительные правила: mhchem для химии, специфичные пакеты LaTeX и т.д.

## 5. Структурные элементы → Markdown

**Боксы «Example / Example Problem»:**

```markdown
!!! example "Пример N.M. Название"
    **Задача.** …

    **Решение.** …

    **Проверь себя.** …

    *Ответ:* …
```

**Заметки/врезки**:

- «Link to Learning» / «Further Reading» → `!!! tip "Дополнительно"`
- «Sidebar» / тематические врезки → `!!! abstract "<название>"`
- «Portrait of …» → `!!! quote "<название>"`
- Контекстуальные заметки → `!!! note "<название>"`

**Конец главы:**

```markdown
## Ключевые термины

`термин`
:   определение.

## Резюме

### N.M Название раздела
Краткий пересказ раздела…

## Упражнения

### N.M Название раздела

1. Вопрос…
2. Вопрос…
```

Нумерацию упражнений сохраняем как в оригинале.

## 6. Рисунки

1. Найди соответствие в `extracted/images/`. Если в `translation/figure_map.json` соответствие уже зафиксировано — используй его. Иначе — открой нужные `extracted/pages/page-NNNN.png`, найди рисунок визуально, сопоставь с `extracted/images/img-NNN.*`, добавь запись в `figure_map.json`.
2. Скопируй файл в `docs/assets/figures/` под именем `fig-CC-NN.ext`, где `CC` — глава, `NN` — номер рисунка.
3. Вставка в Markdown:

```markdown
![Рис. N.M. Краткое описание содержания.](../assets/figures/fig-NN-MM.png)

*Рис. N.M. Полная подпись из оригинала, переведённая на русский.*
```

Альт-текст обязателен, на русском, описывает содержание (для доступности).

## 7. Куда писать результат

Один раздел = один файл. Схема:

```
docs/
└── chapters/
    └── 01-<slug-главы>/
        ├── index.md          # обзор главы
        ├── section-1-1.md
        ├── section-1-2.md
        ├── …
        ├── key-terms.md
        ├── summary.md
        └── exercises.md
```

`<slug-главы>` — короткий kebab-case транслит названия (например, `01-essential-ideas` для «Основные понятия»). Берётся из `chapter_map.json`.

В начале каждого файла-раздела:

```markdown
---
chapter: 1
section: "1.3"
title: "Название раздела"
status: draft           # draft → review → done
source_pages: [28, 35]
source_lines: [1800, 2100]
translator: "Claude/<модель>"
date: YYYY-MM-DD
---

# N.M Название раздела *(Section Title)*

…
```

**Не трогай `mkdocs.yml`** — это работа сборщика.

## 8. Контроль качества — сам себя проверь

Перед тем как объявить раздел готовым, прогони чек-лист:

- [ ] Все формулы оформлены через LaTeX, не Unicode.
- [ ] Десятичная запятая, неразрывные пробелы между числом и единицей.
- [ ] Термины — по `GLOSSARY.md`. Новые добавлены в «Pending review».
- [ ] При первом введении термина — в скобках оригинал.
- [ ] Имена людей проверены по сложившейся традиции и `DOMAIN.md`.
- [ ] Все «Example/Solution/Check Your Learning» переведены и оформлены admonition.
- [ ] Рисунки скопированы, переименованы, с алт-текстом.
- [ ] Подписи к рисункам/таблицам сверены со сканом — числа и формулы совпадают.
- [ ] Нет пропущенных абзацев (сравни число «таблиц/рисунков/примеров» с оригиналом).
- [ ] Frontmatter заполнен, `status: draft`.
- [ ] `translation/PROGRESS.md` обновлён (раздел в `[~]`).
- [ ] Доменные правила из `DOMAIN.md` соблюдены.

## 9. Если ты — агент-ревьюер

Задание: «Проверь раздел N.M».

1. Прочитай переведённый файл и соответствующие `extracted/pages/page-*.png`.
2. Прогони чек-лист §8 — для каждого пункта зафиксируй PASS/FAIL.
3. Сверь термины по `GLOSSARY.md`.
4. Найди пропуски: посчитай число примеров/рисунков/таблиц в оригинале и в переводе.
5. Если всё ок — поменяй `status: draft` на `status: review` в frontmatter и `[~]` → `[r]` в `PROGRESS.md`. Оставь короткий отчёт.
6. Если есть ошибки — НЕ исправляй сам, верни список переводчику.

## 10. Если ты — агент-сборщик

Задание: «Собери главу N».

1. Убедись, что все `section-*.md` главы имеют `status: review` или `done` (и `[r]`/`[x]` в `PROGRESS.md`).
2. Создай/обнови `docs/chapters/<NN-slug>/index.md` — короткое введение из оригинала + автогенерированный список разделов.
3. Запусти `python ../../core/scripts/render_mkdocs.py .` (из `result/<имя>/`) — обновит `nav` в `mkdocs.yml` из `chapter_map.json`.
4. Запусти `mkdocs build --strict`. Любая ошибка/предупреждение — стоп, не коммитить.
5. После успешной сборки переведи `[r]` → `[x]` в `PROGRESS.md` для всех разделов главы.
6. Не делай `git commit`/`push` сам.

---

## 11. Чего НЕ делать (запреты)

- Не переводить и не «улучшать» оригинальные `extracted/` (это входные артефакты).
- Не запускать `git push`, не открывать PR без явной команды.
- Не вызывать `mkdocs gh-deploy`.
- Не редактировать `site/`.
- Не использовать веб-поиск для «уточнения» оригинала — источник истины это PDF и PNG в репозитории.
- Не переводить английские идентификаторы кода (если попадутся блоки кода — оставлять как есть, переводить только комментарии).
- Не вставлять emoji в учебный текст.
- Не делать «креативные» дополнения от себя.
