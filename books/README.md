# books/ — входные PDF

Положи сюда один PDF на одну книгу:

```
books/
├── my-book.pdf
└── another-book.pdf
```

Дальше запусти Claude Code в корне `perevodnik/` и скажи ему:

> «переведи `books/my-book.pdf` на русский»

или, если ты возобновляешь работу:

> «продолжай»

Всё, что генерирует пайплайн (извлечённый текст, постраничные PNG, переведённые Markdown-файлы, конфиг MkDocs, итоговый сайт), окажется в `result/my-book/`. Сама папка `books/` остаётся неизменной — её содержимое только читается.

## Ограничения

- PDF должен содержать **текстовый слой**. Сканы без OCR пайплайн пока не поддерживает (`pdftotext` извлечёт пустой файл). Сначала прогони такой PDF через OCR (например, `ocrmypdf`).
- Имя файла — `kebab-case`, без пробелов и кириллицы. Имя без расширения = имя папки в `result/`.
